Performance monitoring - Photovoltaic plants

My first data engineering project.

The area was struggling filling up all plants data mannualy. It was quite a bit of excel lines to work with.
My mission was to automate this job, but at the beggining no one had an ideia what tasks exactly could be automated.
Bringing up to the table my web development experience, I came up with the most obvious idea: "Let's gather this data in a database and make a report". The question was "How?". Then I started studying data engineering

Data sources and database

The plants have a SCADA system installed. The project started when a discovered a python library that allowed me to consume the SCADA data via HTTP request
It was easy to understand that use this source alone was not enough. The team have already used inverter plataforms to check the performance.
After a little bit of exploration I accomplished to consume this data via HTTP request too. Sadly, there wasn't a place to load all this data

There was a MySQL database used for the web development team, so I asked permission to create a schema for my use case and then start the data modelling

Key-words: Data Engineering, Python, Airflow, MySQL, Power BI
