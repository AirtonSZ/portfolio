Performance monitoring - Photovoltaic plants

My first data engineering project

Data sources and database

The area was struggling filling up all plants data mannualy. It was quite a bit of excel lines to work with. My mission was to automate this job. Bringing up to the table my web development experience, I came up with the most obvious idea: "Let's gather this data in a database and make a report". So I started studying data engineering.

The plants have a SCADA system installed. After a little bit of exploration I accomplished to consume their data via HTTP request and then I needed to load it somewhere. There was a MySQL database used for the web development team, so I asked permission to create a schema for my use case and started the data modelling.

Extracting value from data

It was incredible when the power BI started showing values, but it was easy to notice the bad quality and the lack of visualization flexibility. I changed some aspects of the data model and improved the cleasing rules in python code. I was ready to show to my team the new dashboard for monitoring our photovoltaic plants. After some iterations of feedbacks and new features, the dashboard was showing real value for operational team

My team was receiving good feedbacks about our analysis and naturally my work became famous inside the company. I begin to be included in more projects, what resulted in one monthly report of energy generation and other dashboard to monitor the revenue entry.

That was the story of my first end-to-end data engineering project and why I'm so thrilled to grow in this carreer!!
